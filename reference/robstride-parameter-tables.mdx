# RobStride Runtime Parameter Table

<mintly-toc>

MotorBridge v0.2.8 uses one unified RobStride runtime parameter table from protocol section 4.

</mintly-toc>

MotorBridge keeps the upper-level RobStride API shape shared across RS models. The active runtime parameter table is the protocol section 4 "read/write single parameter" list in the `0x7005..0x702E` range. Older/manual `0x2000` and `0x3000` function-code fields are not split into per-model high-level tables in v0.2.8.

## Runtime Rules

- Runtime parameter read/write uses communication type 17/18 plus the parameter index below.
- Parameter index is little-endian in `byte[0..2]`; the typed value is little-endian in `byte[4..8]`.
- Parameters that must survive power-cycle require type 22 save after writing.
- RobStride `set_zero_position()` keeps the same upper-level API but now writes `zero_sta(0x7029)=1` behind the scenes so zeroed motors use `-pi..pi`. Use `store_parameters` / `--store 1` to persist it.
- Type 22 save uses payload `01 02 03 04 05 06 07 08`.

## Communication Types Used

| Type | Name | Purpose |
|---:|---|---|
| 6 | `SET_ZERO_POSITION` | Set current mechanical position as zero |
| 17 | `READ_PARAMETER` | Read one parameter by index |
| 18 | `WRITE_PARAMETER` | Write one parameter by index |
| 22 | `SAVE_PARAMETERS` | Persist parameter changes |

## Section 4 Parameter List

| Index | Name | Type | Bytes | Unit / Meaning | R/W |
|---:|---|---:|---:|---|---|
| `0x7005` | `run_mode` | `u8` | 1 | `0` MIT, `1` PP, `2` velocity, `3` current, `5` CSP | W/R |
| `0x7006` | `iq_ref` | `f32` | 4 | Current-mode Iq target, `-43..43 A` | W/R |
| `0x700A` | `spd_ref` | `f32` | 4 | Velocity-mode target, `-20..20 rad/s` | W/R |
| `0x700B` | `limit_torque` | `f32` | 4 | Torque limit, `0..60 Nm` | W/R |
| `0x7010` | `cur_kp` | `f32` | 4 | Current-loop Kp, default `0.17` | W/R |
| `0x7011` | `cur_ki` | `f32` | 4 | Current-loop Ki, default `0.012` | W/R |
| `0x7014` | `cur_filter_gain` | `f32` | 4 | Current filter gain, `0..1`, default `0.1` | W/R |
| `0x7016` | `loc_ref` | `f32` | 4 | Position target, `rad` | W/R |
| `0x7017` | `limit_spd` | `f32` | 4 | CSP position speed limit, `0..20 rad/s` | W/R |
| `0x7018` | `limit_cur` | `f32` | 4 | Velocity/position current limit, `0..43 A` | W/R |
| `0x7019` | `mechPos` | `f32` | 4 | Load-side counted mechanical angle, `rad` | R |
| `0x701A` | `iqf` | `f32` | 4 | Filtered iq, `A` | R |
| `0x701B` | `mechVel` | `f32` | 4 | Load-side velocity, `rad/s` | R |
| `0x701C` | `VBUS` | `f32` | 4 | Bus voltage, `V` | R |
| `0x701E` | `loc_kp` | `f32` | 4 | Position-loop Kp, default `60` | W/R |
| `0x701F` | `spd_kp` | `f32` | 4 | Speed-loop Kp, default `6` | W/R |
| `0x7020` | `spd_ki` | `f32` | 4 | Speed-loop Ki, default `0.02` | W/R |
| `0x7021` | `spd_filter_gain` | `f32` | 4 | Speed filter gain, default `0.1` | W/R |
| `0x7022` | `acc_rad` | `f32` | 4 | Velocity-mode acceleration, default `20 rad/s^2` | W/R |
| `0x7024` | `vel_max` | `f32` | 4 | PP mode velocity, default `10 rad/s` | W/R |
| `0x7025` | `acc_set` | `f32` | 4 | PP mode acceleration, default `10 rad/s^2` | W/R |
| `0x7026` | `EPScan_time` | `u16` | 2 | Report period: `1` means `10 ms`, each +1 adds `5 ms` | W |
| `0x7028` | `canTimeout` | `u32` | 4 | CAN timeout; `20000` means `1 s`; default `0` | W |
| `0x7029` | `zero_sta` | `u8` | 1 | `0` = `0..2pi`, `1` = `-pi..pi`; save with type 22 | W |
| `0x702A` | `damper` | `u8` | 1 | Power-off back-drive damping switch; `1` cancels damping | W/R |
| `0x702B` | `add_offset` | `f32` | 4 | Zero offset, `rad`; default `0` | W/R |
| `0x702C` | `alveolous_open` | `u8` | 1 | Cogging compensation enable; default `0` | W/R |
| `0x702D` | `iq_test` | `u8` | 1 | Initialization calibration switch; default `0` | W/R |
| `0x702E` | `dcc_set` | `f32` | 4 | PP-mode deceleration, default `10 rad/s^2` | W/R |

## Examples

Read `mechPos`:

```bash
motorbridge-cli robstride-read-param \
  --channel can0 --model rs-06 --motor-id 2 --feedback-id 0xFD \
  --param-id 0x7019 --type f32
```

Write `damper=1` and save:

```bash
motorbridge-cli robstride-write-param \
  --channel can0 --model rs-06 --motor-id 2 --feedback-id 0xFD \
  --param-id 0x702A --type u8 --value 1 --store 1 --verify 1
```

Set zero through the unified command. For RobStride, MotorBridge also writes `zero_sta=1` internally:

```bash
motorbridge-cli run \
  --vendor robstride --channel can0 --model rs-06 \
  --motor-id 2 --feedback-id 0xFD \
  --mode zero --zero-exp 1 --store 1
```
