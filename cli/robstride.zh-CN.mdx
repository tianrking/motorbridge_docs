# RobStride CLI 子命令

本页是 `motorbridge` 当前 RobStride 控制、参数读写、以及能力边界的完整实用文档。

> English version: [robstride.md](robstride.md)

**相关文档：** [CLI 参考](reference.zh-CN.md) | [shared/channel-compat.md](../shared/channel-compat.md)

## 通道兼容详情

> **通道兼容详情：** 参见 [shared/channel-compat.md](../shared/channel-compat.md)

## 支持模式

| 模式 | 说明 |
|---|---|
| `ping` | 在线探测/握手 |
| `scan` | 范围扫描 |
| `enable` | 使能电机 |
| `disable` | 失能电机 |
| `mit` | MIT 阻抗控制（`pos/vel/kp/kd/tau`） |
| `pos-vel` | 位置速度控制（映射到原生 Position） |
| `vel` | 速度控制 |
| `read-param` | 按参数 ID 读取参数 |
| `write-param` | 按参数 ID 写入参数 |

大一统"四协议"映射状态：

| 大一统能力 | RobStride 状态 | 说明 |
|---|---|---|
| `MIT` | 已支持 | 原生 operation-control 帧 |
| `POS_VEL` | 已支持 | 映射到 `run_mode=1` + `0x7017/0x7016` |
| `VEL` | 已支持 | 映射到 `run_mode=2` + `0x700A` |
| `TORQUE/CURRENT` | 仅参数级 | 尚无统一高层模式；通过 `write-param` 写 `iq_ref`/限幅参数 |

## 1）通用设备参数

| 参数 | 含义 | 常用值 |
|---|---|---|
| `channel` | CAN 接口名 | `can0` |
| `model` | RobStride 型号字符串 | `rs-00`、`rs-06` |
| `motor-id` | 设备 ID | 如 `127` |
| `feedback-id` | 命令帧里的主机/反馈 ID | 常用 `0xFD` |
| `loop` | 周期控制发送次数 | `20`~`100` |
| `dt-ms` | 周期发送间隔 | `20`~`50` |

## 2）Ping

在线探测与电机握手。

```bash
motor_cli \
  --vendor robstride --channel can0 --model rs-06 --motor-id 127 --feedback-id 0xFD \
  --mode ping
```

## 3）扫描（scan）

在指定 ID 范围内扫描 RobStride 电机。

### 参数

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `--start-id` | u16 | `1` | 扫描起始 ID（1..255） |
| `--end-id` | u16 | `255` | 扫描结束 ID（1..255） |
| `--manual-vel` | f32 | `0.2` | 盲探速度（扫描回退） |
| `--manual-ms` | u64 | `200` | 每个 ID 脉冲时长（扫描回退） |
| `--manual-gap-ms` | u64 | `200` | ID 间隔（扫描回退） |

### 扫描行为细节

- 第一阶段：`ping + 参数查询探测`。
- 若全范围无 ping 命中：自动回退到盲探脉冲（观察电机是否转动）。

### 示例

```bash
motor_cli \
  --vendor robstride --channel can0 --model rs-06 --mode scan --start-id 1 --end-id 255
```

## 4）使能 / 失能（enable / disable）

```bash
# 使能
motor_cli \
  --vendor robstride --channel can0 --model rs-06 --motor-id 127 --feedback-id 0xFD \
  --mode enable --loop 1

# 失能
motor_cli \
  --vendor robstride --channel can0 --model rs-06 --motor-id 127 --feedback-id 0xFD \
  --mode disable --loop 1
```

`enable`/`disable` 无额外控制参数。

## 5）MIT 模式

MIT 阻抗控制，使用原生 operation-control 帧。

### 参数

| 参数 | 类型 | 默认值 |
|---|---|---|
| `--pos` | f32 | `0` |
| `--vel` | f32 | `0` |
| `--kp` | f32 | `8` |
| `--kd` | f32 | `0.2` |
| `--tau` | f32 | `0` |

MIT 映射说明（统一接口 -> RobStride 原生）：

- 有效参数：`--pos`、`--vel`、`--kp`、`--kd`、`--tau`（五个都生效）。
- 单位约定：
  - `--pos`：`rad`
  - `--vel`：`rad/s`
  - `--tau`：`Nm`
  - `--kp`、`--kd`：MIT 闭环增益

### 示例

```bash
motor_cli \
  --vendor robstride --channel can0 --model rs-06 --motor-id 127 --feedback-id 0xFD \
  --mode mit --pos 0 --vel 0 --kp 0.5 --kd 0.2 --tau 0 --loop 40 --dt-ms 50
```

## 6）位置模式（统一 `pos-vel` 映射）

位置控制已映射到 RobStride 原生 Position 链路。

### 参数

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `--pos` | f32 | `0` | 目标位置（rad） |
| `--vlim` | f32 | `1.0` | 速度限制（rad/s） |
| `--kp` / `--loc-kp` | f32 | 无（可选） | 位置环 Kp |

说明：

- 统一 `pos-vel` 已映射为 RobStride 原生 Position 链路：
  - `run_mode=1`（Position）
  - 写 `0x7017`（`limit_spd`）为 `--vlim`
  - 可选写 `0x701E`（`loc_kp`）来自 `--loc-kp` 或 `--kp`
  - 写 `0x7016`（`loc_ref`）为 `--pos`
- `--vel`、`--kd`、`--tau` 不属于原生 Position 模式，在 `--mode pos-vel` 下会被忽略（CLI 在传入时会打印 warning）。

### 示例

```bash
motor_cli \
  --vendor robstride --channel can0 --model rs-06 --motor-id 127 --feedback-id 0xFD \
  --mode pos-vel --pos 1.0 --vlim 1.5 --loop 1 --dt-ms 20
```

## 7）速度模式（vel）

速度控制已映射到 RobStride 原生速度链路。

### 参数

| 参数 | 类型 | 默认值 |
|---|---|---|
| `--vel` | f32 | `0` |

### 示例

```bash
motor_cli \
  --vendor robstride --channel can0 --model rs-06 --motor-id 127 --feedback-id 0xFD \
  --mode vel --vel 0.3 --loop 40 --dt-ms 50
```

## 8）读参数（read-param）

按参数 ID 读取单个参数。

### 参数

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `--param-id` | u16 | 必填 | 要读取的参数 ID |

### 示例

```bash
motor_cli \
  --vendor robstride --channel can0 --model rs-06 --motor-id 127 --feedback-id 0xFD \
  --mode read-param --param-id 0x7019
```

## 9）写参数（write-param）

按参数 ID 写入参数。

### 参数

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `--param-id` | u16 | 必填 | 要写入的参数 ID |
| `--param-value` | 类型化值 | 必填 | 按参数元数据解析 |

### 示例

```bash
motor_cli \
  --vendor robstride --channel can0 --model rs-06 --motor-id 127 --feedback-id 0xFD \
  --mode write-param --param-id 0x700A --param-value 0.3
```

## 10）改设备 ID

为电机设置新的设备 ID。

### 参数

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `--set-motor-id` | u16 可选 | 无 | 新设备 ID |
| `--store` | `0/1` | `1` | 保存参数 |

### 示例

```bash
# 旧 ID 127 -> 新 ID 126，并持久化
motor_cli \
  --vendor robstride --channel can0 --model rs-06 \
  --motor-id 127 --feedback-id 0xFD --set-motor-id 126 --store 1
```

与上位机抓包对齐的报文说明：

- 改 ID 使用 `comm_type=7`。
- 该路径下扩展 ID 组成为：
  - `0x07 [new_id] [host_id] [old_id]`
  - 例如（`old_id=1`、`new_id=11`、`host_id=0xFD`）：`0x070BFD01`
- 数据区优先使用最近一次 `ping` 的 UUID token（若拿不到 token 则回退为全 0）。

## 11）零点校准（实验性）

实验性零点校准时序。

```bash
motor_cli \
  --vendor robstride --channel can0 --model rs-00 --motor-id 11 --feedback-id 0xFD \
  --mode zero --zero-exp 1 --store 1
```

注意：RobStride 零点校准仍未稳定。实验性 `zero` 时序可能发送/ACK 正常，但设备侧 `zero_sta`/`mechPos` 校验仍可能失败。在完成固件级时序完全对齐前，零点校准视为未彻底解决。

## 12）两种使用方式（统一封装 / 原生参数）

- **统一封装方式**（推荐上层业务使用）：
  - `--mode mit`
  - `--mode pos-vel`（已映射到原生 Position）
  - `--mode vel`
- **原生方式**（调试/协议级验证）：
  - `--mode read-param --param-id ...`
  - `--mode write-param --param-id ... --param-value ...`
  - 典型链路：先写 `run_mode(0x7005)`，再写对应目标参数（如 `loc_ref/spd_ref`）

## 13）常用参数 ID

| Param ID | 名称 | 类型 | 含义 |
|---|---|---|---|
| `0x7005` | `run_mode` | `i8` | 控制模式选择 |
| `0x700A` | `spd_ref` | `f32` | 目标速度 |
| `0x7019` | `mechPos` | `f32` | 机械位置 |
| `0x701B` | `mechVel` | `f32` | 机械速度 |
| `0x701C` | `VBUS` | `f32` | 母线电压 |

## 14）Python Binding 示例

```python
from motorbridge import Controller

with Controller("can0") as ctrl:
    m = ctrl.add_robstride_motor(127, 0xFD, "rs-06")
    print(m.robstride_ping())
    print(m.robstride_get_param_f32(0x7019, 500))
    m.robstride_write_param_f32(0x700A, 0.3)
    m.close()
```

## 15）协议通信类型覆盖情况

当前 `motorbridge` 对 RobStride 协议通信类型的覆盖：

- **已直接使用：** `0(GET_DEVICE_ID)`、`1(OPERATION_CONTROL)`、`3(ENABLE)`、`4(DISABLE)`、`6(SET_ZERO_POSITION)`、`7(SET_DEVICE_ID)`、`17(READ_PARAMETER)`、`18(WRITE_PARAMETER)`、`22(SAVE_PARAMETERS)`
- **已接收解析：** `2(OPERATION_STATUS)`、`21(FAULT_REPORT)`
- **协议常量存在但尚未形成高层 API：** `23(SET_BAUDRATE)`、`24(ACTIVE_REPORT)`、`25(SET_PROTOCOL)`

## 17）已知问题

1. **`pos-vel` 参数生效性**在部分固件上不稳定：
   - `--vlim`（`0x7017`）和 `--kp`/`loc_kp`（`0x701E`）可能回读正常，但体感效果弱或不明显。
   - 当前 `MIT` 路径相对更稳定。
2. **RobStride 零点校准**仍未稳定：
   - 实验性 `zero` 时序可能发送/ACK 正常，但设备侧 `zero_sta`/`mechPos` 校验仍可能失败。
   - 在完成固件级时序完全对齐前，零点校准视为未彻底解决。

## 18）安全建议

- 先小速度、小循环验证，再逐步增大。
- 压测前先确认 CAN 接线、终端电阻和接口状态。
- 长时间控制前先做 ping/读参验证。
- 始终保留急停路径。
