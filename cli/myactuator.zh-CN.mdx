# MyActuator CLI 子命令

本页是 `motor_cli` 中 MyActuator 控制能力的实用总表。

> English version: [myactuator.md](myactuator.md)

**相关文档：** [CLI 参考](reference.zh-CN.md) | [shared/channel-compat.md](../shared/channel-compat.md)

## 通道兼容详情

> **通道兼容详情：** 参见 [shared/channel-compat.md](../shared/channel-compat.md)

## 1）基础信息

- 厂商名：`myactuator`
- CAN ID 规则：
  - 发送命令：`0x140 + motor_id`
  - 接收反馈：`0x240 + motor_id`
- 常见 ID 范围：`motor_id in [1, 32]`
- 对于 ID=1 的默认值：
  - `--motor-id 1`
  - `--feedback-id 0x241`

## 2）`motor_cli` 支持模式

| 模式 | 协议命令 | 说明 |
|---|---|---|
| `scan` | - | 扫描 1..32 |
| `enable` | `0x77` | 释放刹车 |
| `disable` | `0x80` | 关电机 |
| `stop` | `0x81` | 停止闭环 |
| `set-zero` | `0x64` | 将当前位置写为编码器零点（断电重启后持久生效） |
| `status` | `0x9C` | 读取状态2 |
| `current` | `0xA1` | 电流闭环 |
| `vel` | `0xA2` | 速度闭环 |
| `pos` | `0xA4` | 绝对位置闭环 |
| `version` | `0xB2` | 读取版本日期 |
| `mode-query` | `0x70` | 读取系统运行模式 |

## 3）扫描（scan）

扫描 1..32 范围内的 MyActuator 电机。

```bash
motor_cli --vendor myactuator --channel can0 --mode scan --start-id 1 --end-id 32
```

## 4）使能 / 失能（enable / disable）

```bash
# 使能（释放刹车）
motor_cli --vendor myactuator --channel can0 --model X8 --motor-id 1 --feedback-id 0x241 \
  --mode enable --loop 1

# 失能（关电机）
motor_cli --vendor myactuator --channel can0 --model X8 --motor-id 1 --feedback-id 0x241 \
  --mode disable --loop 1

# 停止闭环
motor_cli --vendor myactuator --channel can0 --model X8 --motor-id 1 --feedback-id 0x241 \
  --mode stop --loop 1
```

## 5）状态查询（status）

连续请求状态2（`0x9C`）。

```bash
motor_cli --vendor myactuator --channel can0 --model X8 --motor-id 1 --feedback-id 0x241 \
  --mode status --loop 40 --dt-ms 50
```

CLI 的状态输出同时打印：

- `angle=...`：来自状态2（`0x9C`）-- 近圈角
- `mt_angle=...`：来自多圈角（`0x92`）-- 用于绝对位置判定

## 6）速度模式（vel）

速度闭环（`0xA2`）。

```bash
# +0.5236 rad/s (~+30 deg/s)
motor_cli --vendor myactuator --channel can0 --model X8 --motor-id 1 --feedback-id 0x241 \
  --mode vel --vel 0.5236 --loop 100 --dt-ms 20
```

## 7）位置模式（pos）

绝对位置闭环（`0xA4`）。

```bash
# pi rad = 180 度，最大 5.236 rad/s ~= 300 deg/s
motor_cli --vendor myactuator --channel can0 --model X8 --motor-id 1 --feedback-id 0x241 \
  --mode pos --pos 3.1416 --max-speed 5.236 --loop 1
```

## 8）电流模式（current）

电流闭环（`0xA1`）。

```bash
motor_cli --vendor myactuator --channel can0 --model X8 --motor-id 1 --feedback-id 0x241 \
  --mode current --current 1.0 --loop 100 --dt-ms 20
```

## 9）设零（set-zero）

将当前位置设为零点。持久生效需要断电重启。

```bash
motor_cli --vendor myactuator --channel can0 --model X8 --motor-id 1 --feedback-id 0x241 \
  --mode set-zero --loop 1
```

## 10）版本和模式查询

```bash
# 读取固件版本日期
motor_cli --vendor myactuator --channel can0 --model X8 --motor-id 1 --feedback-id 0x241 \
  --mode version --loop 1

# 读取系统运行模式
motor_cli --vendor myactuator --channel can0 --model X8 --motor-id 1 --feedback-id 0x241 \
  --mode mode-query --loop 1
```

## 11）参数与单位

| 参数 | 类型 | 默认值 | 作用模式 | 单位 |
|---|---|---|---|---|
| `--start-id` | u16 | `1` | `scan` | id |
| `--end-id` | u16 | `32` | `scan` | id |
| `--current` | f32 | `0.0` | `current` | A |
| `--vel` | f32 | `0.0` | `vel` | rad/s |
| `--pos` | f32 | `0.0` | `pos` | rad |
| `--max-speed` | f32 | `8.726646` | `pos` | rad/s |
| `--loop` | u64 | `1` | 全部 | 次 |
| `--dt-ms` | u64 | `20` | 全部 | ms |

## 12）当前 CLI 的编码细节

CLI 输入统一使用弧度/弧度每秒，再在内部转换为 MyActuator 协议使用的角度/角度每秒。

- `A1` 电流模式：
  - `[4..5] = int16(current / 0.01)`
- `A2` 速度模式：
  - `[4..7] = int32((vel_rad_s.to_degrees()) * 100)`
- `A4` 绝对位置模式：
  - `[2..3] = uint16(max_speed_rad_s.to_degrees())`
  - `[4..7] = int32(pos_rad.to_degrees() * 100)`

## 13）反馈解码（状态/命令回包）

对于 `0x9C`、`0xA1`、`0xA2`、`0xA4` 回包：

- `data[1]`：温度（int8, deg C）
- `data[2..3]`：电流（int16 * 0.01 A）
- `data[4..5]`：速度（int16, deg/s）
- `data[6..7]`：轴角（int16 * 0.01 deg，近圈角）

对于 `0x92` 多圈角回包：

- `data[4..7]`：多圈角（int32 * 0.01 deg）

CLI 的状态输出现在同时打印：

- `angle=...`：来自状态2（`0x9C`）
- `mt_angle=...`：来自多圈角（`0x92`），用于绝对位置判定

## 14）故障排查

如果"有回包但不转"：

1. 用 `0x9A` 查询状态1并看故障码。
2. 若故障码为 `0x0004`，表示**欠压保护**。
3. 先恢复供电电压，再执行复位（`0x76`）与释放刹车（`0x77`）。

状态1常用字段：

- `data[3]`：刹车释放标志
- `data[4..5]`：母线电压（`uint16 * 0.1 V`）
- `data[6..7]`：故障码
