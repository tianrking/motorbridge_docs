# MotorBridge 0.2.7 Release Test Note

Date: 2026-05-12
Scope: RobStride model-specific parameter tables, Rust core/CLI, websocket gateway, Python binding/CLI, and documentation alignment.

## Summary

MotorBridge 0.2.7 makes RobStride parameter access model-aware across RS00-RS06. High-level control command shapes remain the same, but parameter read/write must use the physical motor model because the manual function-code tables differ by model.

## Version Alignment

| Component | Version |
|---|---|
| Git release tag | `v0.2.7` |
| Rust workspace crates | `0.2.4` |
| Python package | `0.2.7` |

## Required RobStride Rule

Always pass the physical RobStride model:

- `rs-00`
- `rs-01`
- `rs-02`
- `rs-03`
- `rs-04`
- `rs-05`
- `rs-06`

Do not use `rs-00` as a generic placeholder for other RS motors. The complete runtime parameter tables are documented in [RobStride Parameter Tables](/reference/robstride-parameter-tables).

## Safe Bench Checks

Use the real model installed on the bench motor.

```bash
cargo run -p motor_cli --release -- robstride-read-param \
  --channel can0 --model rs-06 --motor-id 2 --feedback-id 0xFD \
  --param-id 0x7019 --type f32
```

```bash
motorbridge-cli robstride-read-param \
  --channel can0 --model rs-06 --motor-id 2 --feedback-id 0xFD \
  --param-id 0x7019 --type f32
```

Expected:

- CLI output says `vendor=robstride`.
- CLI output shows the selected physical model, for example `model=rs-06`.
- `0x7019` resolves as `mechPos` and returns an `f32` value.
