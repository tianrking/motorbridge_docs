# MotorBridge 0.2.8 Release Test Note

Date: 2026-05-12
Scope: v0.2.6-compatible unified interfaces, RobStride protocol section 4 runtime parameters, zeroing behavior, Rust core/CLI, websocket gateway, Python binding/CLI, and documentation alignment.

## Summary

MotorBridge 0.2.8 restores the unified v0.2.6 RobStride command shape and applies the minimal protocol fixes needed for production use. RobStride parameter read/write uses the common protocol section 4 runtime list `0x7005..0x702E`. The upper-level zero command is unchanged, but RobStride now writes `zero_sta(0x7029)=1` behind the scenes so zeroed motors use the `-pi..pi` startup range.

## RobStride Checks

Read runtime position:

```bash
cargo run -p motor_cli --release -- robstride-read-param \
  --channel can0 --model rs-06 --motor-id 2 --feedback-id 0xFD \
  --param-id 0x7019 --type f32
```

Write one of the newly completed section 4 parameters:

```bash
motorbridge-cli robstride-write-param \
  --channel can0 --model rs-06 --motor-id 2 --feedback-id 0xFD \
  --param-id 0x702A --type u8 --value 1 --store 1 --verify 1
```

Zero through the unchanged upper-level command:

```bash
motorbridge-cli run \
  --vendor robstride --channel can0 --model rs-06 \
  --motor-id 2 --feedback-id 0xFD \
  --mode zero --zero-exp 1 --store 1
```

Expected:

- `set_zero_position` sends type 6, then writes `zero_sta(0x7029)=1` internally.
- `--store 1` / `store_parameters` sends type 22 with payload `01 02 03 04 05 06 07 08`.
- No websocket command shape changes are required.
