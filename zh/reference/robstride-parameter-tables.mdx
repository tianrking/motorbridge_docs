# RobStride 参数表

<mintly-toc>

MotorBridge 使用的 RobStride RS00-RS06 完整功能码/参数表文本版，便于查询、校验和现场调试。

</mintly-toc>

来源对齐：RobStride/Product_Information commit [`ba7236bc26417766fda71e75ae128c66dbd21aba`](https://github.com/RobStride/Product_Information/commit/ba7236bc26417766fda71e75ae128c66dbd21aba)。下面表格由 MotorBridge core 文件 `motor_vendors/robstride/src/registers_00.rs` 到 `registers_06.rs` 生成，因此文档和运行时编码/解码使用同一套 ID 与数据类型。

## 使用规则

- 必须传真实电机型号：`rs-00`、`rs-01`、`rs-02`、`rs-03`、`rs-04`、`rs-05` 或 `rs-06`。
- 不要把 `rs-00` 当作所有 RobStride 电机的通用默认值。
- 读写参数时，`--param-id` 和 `--type` 必须和当前型号表一致。
- `0x7000` 段是统一控制路径使用的公共运行参数；`0x2000` 和 `0x3000` 手册参数段会随型号变化。
- String 项保留在表里方便查阅；CLI 目前有意不支持写 string 参数。
- Rust `motor_cli` 使用完整表类型，包括 `i16` 和 `i32`。Python `motorbridge-cli` 0.2.7 暴露的是当前 ABI 参数类型：`i8`、`u8`、`u16`、`u32`、`f32`；如果要读写表里的 `i16`/`i32` 项，请使用 Rust CLI/core 路径。

## 命令示例

读取参数：

```bash
motorbridge-cli robstride-read-param \
  --channel can0 --model rs-06 --motor-id 2 --feedback-id 0xFD \
  --param-id 0x7019 --type f32
```

写参数并回读校验：

```bash
motorbridge-cli robstride-write-param \
  --channel can0 --model rs-06 --motor-id 2 --feedback-id 0xFD \
  --param-id 0x7017 --type f32 --value 2.0 --verify 1
```
